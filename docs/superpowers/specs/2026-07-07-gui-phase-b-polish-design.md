# GUI Phase B — UX polish to product grade (design)

Дата: 2026-07-07 (сессия 22). Утверждено Mr.Di в брейншторме.

## Цель

Довести существующие GUI-агенты (`gui/macos/ParanoidBar.swift`, `gui/windows/paranoid-tray.ps1`)
до «превосходного продукта» для нетехнической аудитории — **без дистрибуции** (подпись/нотаризация —
отдельный этап) и **без нарушения архитектурного принципа**: GUI секретов не держит, крипту не
добавляет, только запускает те же подписанные CLI; вывод и ввод секретов — в терминале.

Архитектура не меняется: single-file Swift (компилируется `swiftc`, без Xcode-проекта) и
single-file PowerShell. Новых зависимостей нет. Полный паритет фич macOS ↔ Windows.

## 1. Глобальный хоткей паники

Ядро ценности: в реальной панике набор команды бесполезен — нужна кнопка из любого приложения.

- **macOS:** Carbon `RegisterEventHotKey`. Дефолт **⌃⌥⇧P**.
- **Windows:** `RegisterHotKey` через P/Invoke (`Add-Type`). Дефолт **Ctrl+Alt+Shift+P**.
- **Двойное нажатие в течение 2 секунд** → немедленно `panic now` (терминал/консоль открывается,
  вывод виден — контракт честности сохранён; confirm-диалога нет: `panic now` обратим — прячет
  и лочит, данные не уничтожает).
- Первое нажатие → уведомление «Press again to PANIC» + мигание глифа/иконки.
- Настройка в Settings: пресеты (⌃⌥⇧P / ⌃⌥⇧L / Off), не произвольный key-recorder
  (проще и надёжнее в single-file). Off = хоткей не регистрируется.

ASSUMPTION (зафиксировано в брейншторме): double-press без confirm — баланс
«защита от случайного нажатия» / «скорость реальной паники».

## 2. Уведомления

Read-only поверх существующего poll-цикла, никаких новых демонов, не спамить (каждое событие —
один раз за эпизод).

- **macOS:** `osascript -e 'display notification …'` — работает из неподписанного бинаря
  (`UNUserNotificationCenter` требует bundle — не наш случай).
- **Windows:** `NotifyIcon.ShowBalloonTip`.

События:
1. TTL vaultwatch < 2 мин → «vault авто-закроется через N».
2. TTL истёк, vault всё ещё открыт → предупреждение.
3. Vault открыт дольше 30 мин без vaultwatch-сессии → мягкое напоминание (однократно за эпизод
   открытия).
4. Первый тик panic-хоткея → «Press again to PANIC».

## 3. Онбординг first-run (вариант A — Welcome-окно)

Показывается один раз при первом запуске (флаг в UserDefaults / HKCU); всегда доступно из меню
(«Setup guide»). Одно окно, живой чеклист готовности с кнопками-действиями:

- ✅/❌ CLI установлены (securetrash, panic, vaultwatch — проверка `command -v` / `Get-Command`);
  при ❌ — кнопка открывает страницу установки (README).
- ✅/❌ Сейф создан → кнопка «Create vault…» (запускает `securetrash vault create` в терминале).
- ⬜ Хоткей паники → кнопка «Enable» (включает дефолтный пресет).
- ⬜ Автостарт при входе → кнопка «Enable» (существующий LaunchAgent / HKCU Run).

Обязательная строка честности в окне: открытый сейф всегда «at risk» — GUI это не прячет.

## 4. Локализация RU/EN

- Словарь строк в коде: Swift — `L("key")` + таблица RU/EN; ps1 — hashtable. Без .lproj-бандлов —
  single-file принцип сохраняется.
- Выбор: системная локаль → override в Settings («Language: System / English / Русский»).
- Локализуется только GUI-обвязка; вывод CLI остаётся английским (территория самих тулов).
- Честные формулировки («at risk», Scope) переводятся без смягчения.
- Паритет ключей RU=EN и macOS↔Windows проверяется тестом.
